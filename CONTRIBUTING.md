# Contributing to Genius

Thank you for your interest in contributing to Genius! I welcome contributions from the community, whether it's reporting bugs, suggesting features, improving documentation, or submitting code changes. All contributions are valued and help make this plugin better for everyone.

## How Can I Contribute?

### Reporting Bugs

- Before creating a bug report, please check the [existing issues](https://github.com/fletchly/genius/issues) to see if the problem has already been reported.
- If you're unable to find an open issue, [open a new one](https://github.com/fletchly/genius/issues/new?template=bug_report.md).
- Be sure to include a clear title and description, as much relevant information as possible (e.g., Minecraft version, Paper version, steps to reproduce, expected vs. actual behavior), and any error logs or screenshots.

### Suggesting Features or Enhancements

- Feature requests are welcome! Open an [issue](https://github.com/fletchly/genius/issues/new?template=feature_request.md) with a clear description of the proposed feature, why it would be useful, and any examples or mockups if applicable.

### Pull Requests

- Fork the repository and create a new branch for your changes (e.g., `feat/my-new-feature` or `fix/bug-description`).
- Follow the existing code style (we use standard Kotlin conventions)
- Write clear, concise commit messages.
- Ensure your code compiles and passes any existing tests.
- Add or update tests if you're adding new functionality or fixing a bug.
- Update documentation (e.g., README.md, plugin.yml descriptions) as needed.
- Target the latest supported Minecraft/Paper version unless specified otherwise.
- Open a [pull request](https://github.com/fletchly/genius/compare) with a clear description of changes and reference any related issues.
